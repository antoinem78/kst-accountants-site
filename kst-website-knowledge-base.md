## 1. Project at a glance

KST Accountants is a chartered certified accountancy firm in Buckhurst Hill, Essex. The brief was to replace their existing Weebly site with a modern, conversion-optimised marketing site that:

- Looks credible and modern (matches the navy / gold / teal of the existing logo).
- Has a single, prominent CTA on every page (`Book a consultation`).
- Captures leads into GoHighLevel (their CRM) via a server API route.
- Preserves the existing Weebly tracking IDs so analytics history isn't lost.
- Is easy for a non-developer to edit (copy lives in plain TypeScript files).

The five service lines are: Accountancy, Taxation, VAT, Payroll, CIS.

Owner contact at KST: **Kris**, `Kris@kst-accountants.co.uk`.

## 4. Environment variables and secrets

All env vars live in `.env.local` (git-ignored). Template in `.env.example`.

| Variable | Required | Where used | Value |
|---|---|---|---|
| `NEXT_PUBLIC_SITE_URL` | yes | metadata, sitemap, OG | `https://www.kst-accountants.co.uk` |
| `GHL_API_KEY` | yes (prod) | `app/api/lead/route.ts` (server-only) | from GHL Private Integration |
| `GHL_LOCATION_ID` | yes (prod) | `app/api/lead/route.ts` | from GHL sub-account |
| `NEXT_PUBLIC_GA_ID` | optional | `app/layout.tsx` | `G-1S1VWVV2SE` |
| `NEXT_PUBLIC_GTM_ID` | optional | `app/layout.tsx` | `GTM-TT4QNR3G` |

Rules:

- Anything prefixed `NEXT_PUBLIC_` is bundled into the browser. Never put secrets behind that prefix.
- `GHL_API_KEY` and `GHL_LOCATION_ID` are server-only. They're read inside `app/api/lead/route.ts` (`runtime = 'nodejs'`).
- If `GHL_API_KEY`/`GHL_LOCATION_ID` are missing, the lead route logs the lead to stdout and returns `{ ok: true, forwarded: false }` — the form still "works" in dev.

In Vercel, these go in **Project → Settings → Environment Variables**, scoped to **Production** (and **Preview**, if you want lead capture from preview deploys to land in GHL).

## 7. Vercel deployment workflow

### First-time setup

1. **Create the project**
   - Go to <https://vercel.com/new>.
   - Pick the GitHub repo `kst-accountants-site`.
   - Framework preset auto-detects **Next.js** — leave defaults (Build = `next build`, Output = `.next`, Install = `npm install`).

2. **Set env vars** (before clicking Deploy)
   - `NEXT_PUBLIC_SITE_URL=https://www.kst-accountants.co.uk`
   - `GHL_API_KEY=<from GHL>`
   - `GHL_LOCATION_ID=<from GHL>`
   - `NEXT_PUBLIC_GA_ID=G-1S1VWVV2SE`
   - `NEXT_PUBLIC_GTM_ID=GTM-TT4QNR3G`

3. **Deploy.** First build takes ~90 seconds. The preview URL is `kst-accountants-site.vercel.app` (or similar).

### Ongoing deploys

- Push to `main` → Vercel auto-builds and deploys to production within ~60 seconds.
- Open a PR → Vercel creates a Preview deployment with its own URL; comment posted on the PR.
- Roll back: Vercel **Deployments → … → Promote** the previous one.

### Project settings worth knowing

- **Functions region.** `app/api/lead/route.ts` runs as a Node serverless function. Default region (`iad1`) is fine; change to `lhr1`/`fra1` to be closer to UK users.
- **Edge vs Node.** The lead route is pinned `runtime = 'nodejs'` because the GHL fetch and the in-memory rate limiter are simpler there. If you ever switch to Edge, replace the `Map`-based rate limiter with Upstash/Redis.
- **Build cache.** Vercel caches `node_modules` between builds; force a clean build with **Redeploy → without cache** when dependencies look weird.

## 9. GoHighLevel (CRM) integration

### Where leads land

`POST /api/lead` (defined in `app/api/lead/route.ts`) is called by `LeadForm.tsx`. The route:

1. Rate-limits by IP (5 requests / 60s, in-memory).
2. Validates the JSON body against the Zod `leadSchema` in `lib/validation.ts`.
3. Honeypots — if the hidden `website` field is filled, returns `{ok: true}` silently (bot bait).
4. If `GHL_API_KEY` and `GHL_LOCATION_ID` are unset, logs the lead and returns success.
5. Otherwise upserts into GHL with `Authorization: Bearer <key>`, `Version: 2021-07-28`.
6. On non-2xx, returns 502 with a fallback message including the office phone.

### Payload sent to GHL

```json
{
  "locationId": "...",
  "firstName": "...",
  "lastName": "...",
  "email": "...",
  "phone": "...",
  "companyName": "(optional)",
  "source": "kst-accountants.co.uk · <pageSource>",
  "tags": ["website-lead", "service:vat"],
  "customFields": [
    { "key": "service_of_interest", "field_value": "VAT" },
    { "key": "enquiry_message", "field_value": "..." },
    { "key": "source_page", "field_value": "services/vat" }
  ]
}
```

### Setup steps in GHL

1. **Settings → Private Integrations → Create new.** Scopes: `contacts.write`, `contacts.readonly`. Copy the token → `GHL_API_KEY`.
2. **Settings → Business Profile** (or sub-account URL): copy the location ID → `GHL_LOCATION_ID`.
3. **Custom fields** (Settings → Custom Fields → Contacts): create
   - `service_of_interest` (Single Line)
   - `enquiry_message` (Multi Line)
   - `source_page` (Single Line)
4. **Workflows:** trigger on `Contact Created with tag = website-lead` → email Kris, add to nurture pipeline, send SMS, etc.

### Testing the integration

```bash
curl -X POST https://www.kst-accountants.co.uk/api/lead \
  -H "Content-Type: application/json" \
  -d '{
    "firstName":"Test","lastName":"User","email":"test@example.com",
    "phone":"0203 150 2074","service":"VAT","message":"hi","consent":true,
    "pageSource":"manual-test"
  }'
```

Expect `{"ok":true,"forwarded":true}` and a contact in GHL within a couple of seconds.

## 10. Tracking — GTM and GA4

Both are loaded in `app/layout.tsx` via `next/script` with `strategy="afterInteractive"` so they don't block paint.

- **GTM** ID: `GTM-TT4QNR3G`. The script tag is in `<head>`; the `<noscript>` iframe fallback is in `<body>`.
- **GA4** measurement ID: `G-1S1VWVV2SE`. Loaded as a separate gtag.js script alongside GTM (belt-and-braces: GA continues working even if GTM is misconfigured).
- **dataLayer events:** `LeadForm.tsx` pushes `{ event: 'lead_submit' }` on a successful submission. Use this in GTM to fire a conversion tag (Google Ads conversion, GA4 event, etc.).

If you want server-side tracking later, the Anthropic-recommended pattern is to fire from `/api/lead/route.ts` to GA's Measurement Protocol after the GHL upsert succeeds — keep that in mind, not implemented yet.

## 12. Design system — colour, type, components

### Colours (`tailwind.config.ts`)

The class names retain the `navy-*` prefix for backward compatibility, but the actual values map to the **teal scale** from the live KST logo. Two palettes plus neutrals:

- `navy` (teal): `#5199A8` is the brand mid-tone (`navy-500`); deep `navy-900` `#1A3A44` is used on backgrounds and headings.
- `gold`: accent. `gold-500` `#D4A017` is the chart-icon gold from the logo.
- `ink`: copy colours — `#222222` default, `#484848` muted, `#8E8E8E` soft.
- `bg`: page background `#F9F9F9`, subtle muted `#F1F5F5`.
- Custom shadows: `shadow-soft`, `shadow-lift`.
- Animations: `animate-fade-in`, `animate-fade-in-up`.

### Typography

- **Body:** Inter (`--font-inter`), system fallback.
- **Display/headings:** Fraunces (`--font-fraunces`), Georgia fallback.
- Both loaded via `next/font/google` with `display: swap` — no FOIT, no render-blocking CSS.

### Components — at a glance

- **Header.** Sticky, transparent at top, becomes solid on scroll. Single primary CTA: `Book a consultation`. Phone number is click-to-call.
- **HeroSlider.** Rotating headlines/subheads on the homepage, ~6.5s interval, pauses on hover. Background gradient layered with optional image.
- **CredentialsStrip.** Logos/text strip — ACCA Chartered Certified, HMRC Agent, ICO Registered.
- **Service tiles (homepage).** 3-column grid, lifts on hover, gold accent on hover.
- **NumericUSP.** Four animated count-up stats: 500+ clients, 15+ years, 98% retention, 100% on-time filing.
- **WhyChooseUs.** 6-card benefit grid above the fold.
- **FAQ.** Accordion driven by `lib/faq.ts`; emits `FAQPage` JSON-LD.
- **Testimonials.** Cards with 5-star ratings; emits `AggregateRating` JSON-LD.
- **LeadForm.** Reusable across homepage / `/contact` / each service page. Compact mode for the inline service-page version. Honeypot field. Success state with thank-you message.
- **Footer.** Services, contact details, credentials, social, copyright.
- **JsonLd.** Three exported components: `OrganizationJsonLd`, `FaqJsonLd`, `ReviewsJsonLd`.

## 14. Lead-form pipeline (end-to-end)

```
┌───────────────┐
│  LeadForm.tsx │  fetch('/api/lead', {method:'POST', body: JSON})
└──────┬────────┘
       │
       ▼
┌──────────────────────────────────────────────────────────┐
│  app/api/lead/route.ts  (Node runtime)                   │
│  1. rate-limit by IP (5/min)                             │
│  2. zod validate (firstName/lastName/email/phone/        │
│     consent required; service/message/company optional;  │
│     honeypot 'website' must be empty)                    │
│  3. if honeypot tripped → return ok silently             │
│  4. if GHL env unset → log + return ok (dev mode)        │
│  5. else POST to services.leadconnectorhq.com/contacts/  │
│     upsert with Bearer GHL_API_KEY                       │
└──────┬───────────────────────────────────────────────────┘
       │
       ▼
┌───────────────────────────┐
│  GoHighLevel (CRM)        │
│  Workflow: tag=website-   │
│  lead → email/SMS Kris,   │
│  add to nurture sequence. │
└───────────────────────────┘
```

Validation rules (`lib/validation.ts`):

- `firstName`, `lastName`: 1–60 chars, trimmed.
- `email`: RFC email, max 120.
- `phone`: 7–30 chars, regex `^[0-9+\-\s()]+$`.
- `company`, `service`, `message`: optional (≤120, ≤60, ≤2000).
- `consent`: must literally be `true`.
- `website` (honeypot): must be empty.
- `pageSource`: optional, ≤200.

## 16. Operational runbook

### Post-deploy smoke checks

After every production deploy that touches anything material:

- Homepage loads; hero slider rotates.
- Phone `020 3150 2074` is in header/footer and is a `tel:` link on mobile.
- All five service pages render: `/services/accountancy`, `/taxation`, `/vat`, `/payroll`, `/cis`.
- `/about`, `/contact`, `/privacy` render.
- Submit a test lead from `/contact` → contact appears in GHL within ~2s.
- `/sitemap.xml` and `/robots.txt` reachable.
- GTM Preview mode fires the pageview for `GTM-TT4QNR3G`.
- GA4 Realtime shows the visit under `G-1S1VWVV2SE`.

### Common issues

| Symptom | Likely cause | Fix |
|---|---|---|
| Form returns 502 | GHL API rejected the request | Check Vercel function logs; verify `GHL_API_KEY` not rotated; test endpoint with curl |
| Form returns 429 | More than 5 submits from same IP in 60s | Wait or relax limit in `route.ts` |
| Form submits but no contact in GHL | Honeypot was tripped (bot?) or env vars unset on this Vercel environment | Check Production vs Preview env vars; check function logs for honeypot warning |
| Build fails on Vercel | Type error or missing dep | Reproduce with `npm run build` locally; commit fix |
| GA4 Realtime empty | Adblocker or `NEXT_PUBLIC_GA_ID` unset | Test in incognito; verify env var is set and prefix is `NEXT_PUBLIC_` |
| Wrong colours after Tailwind change | Tailwind purge missed the file | Add path to `content` in `tailwind.config.ts` |

### Rotating the GHL API key

1. In GHL, create a new Private Integration with the same scopes.
2. Update `GHL_API_KEY` in Vercel env vars (Production + Preview).
3. Redeploy (or wait for next push).
4. Revoke the old integration in GHL.

### Backups

- Code: GitHub is the source of truth.
- Env vars: store the production values in 1Password / Bitwarden vault. Vercel does not show the value back to you once entered.
- Lead data: lives in GHL — back it up from GHL's contact export, not from this site.

7. **Read `app/layout.tsx`** — GTM/GA4 + global metadata + skip-link.
8. **Run locally:** `npm install && npm run dev`. Browse all routes.
9. **Check git state.** If `.git/` is missing, initialise per §6.
10. **Confirm pending items** (§17) with the user before assuming launch readiness.

Common failure modes to avoid:

- Don't add a UI library (shadcn, MUI, Chakra). The project deliberately uses raw Tailwind + native HTML to keep the bundle small.
- Don't move secrets into `NEXT_PUBLIC_*`. `GHL_API_KEY` must stay server-only.
- Don't change the GTM or GA4 IDs without checking with Antoine — they were ported from Weebly to preserve historical analytics.
- Don't introduce a new font without `display: swap`. Performance budget is tight.
- Don't replace the in-memory rate limiter on a serverless deployment unless you actually need it — Vercel's per-instance state is fine for this traffic level.
